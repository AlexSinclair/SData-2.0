SData 2.0 on GitHub

Welcome !

This repository provides technical information on the upcoming version 2.0 of the standard.

SData is a web toolkit, promoting the development of REST-orientated services and their consumption. It consists of a small, standards-based core (HTTP, ATOM, JSON) and a set of optional mechanisms solving common implementation challenges in a consistent manner.

The version 2.0 of the standard delivers additions aimed towards the needs of web product- and interface definitions. The focal points of the new release are:

  • Specification of the JSON format for SData information exchanges
  • Relaxation of several SData restrictions, resulting in a smaller, better-defined specification core
  • Compatibility with the SData 1.1 version, ensuring that SData 1.1 implementations are automatically version 2.0 compliant
  • Clarification of the role and form of contracts
  • Reorganization of the standard, stressing the freedom of choice for development teams and architects
 

The documents contained in our GitHub file repository preview a number of important aspects of the standard's upcoming release. They are to be considered working drafts, expanding and ammending the SData 1.1 specification. We will continue to update and extend the information as we make progress towards the release.

The following documents pertaining to SData 2.0 are available (click on the links to download directly):

  • Approach: The SData 2.0 approach document presents a number of key statements and goals that guide the current development of the standard
  • Contracts: The Contracts document describes the concept of an SData contract, an area not clearly presented in the current version of the standard. It shows the role of contracts, describes the many shapes they may take and finishes by outlining some items that could be considered by contract designers.
  • Core: The Core document outlines the essential components of SData 2.0 and describes the differences to the 1.1 version of the standard.
  • JSON Formatted responses: The document describes the various aspects relating to the integration of JSON in SData. It covers the response JSON formatting as well as the mechanisms used to request JSON from a provider.

The documents are hosted at https://github.com/Sage/SData-2.0, our file repository. We welcome your visit!

You may also find these SData-related GitHub repositories of interest:

  • Argos-template: template project for Argos

  • SDataCSharpClientLib: C# library aiding the consumption of SData services
